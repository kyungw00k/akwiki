# 마크다운 문법

akwiki에서 사용하는 마크다운 문법을 정리합니다.

## Frontmatter

각 페이지 상단에 YAML frontmatter를 작성할 수 있습니다.

```yaml
---
title: 페이지 제목
titleKo: 한국어 제목
type: Article
private: false
aliases:
  - 다른 이름
tags:
  - 태그
---
```

모든 필드는 선택입니다. `title`이 없으면 파일명이 제목이 됩니다.

| 필드 | 기본값 | 설명 |
|------|--------|------|
| title | 파일명 | 영문 제목 |
| titleKo | - | 한국어 제목 (표시 우선) |
| type | Article | Article, Book, ScholarlyArticle, Journal |
| private | false | true면 빌드에서 완전 제외 |
| aliases | [] | 이 이름으로도 위키링크 가능 |
| tags | [] | 분류 태그 |

## 위키링크

페이지 간 연결은 `[[` `]]` 문법을 사용합니다.

```markdown
[[페이지 이름]]              → 해당 페이지로 링크
[[페이지 이름|표시 텍스트]]   → 표시 텍스트로 링크
```

예시: [[Home|홈으로 돌아가기]], [[위키링크]]

## 일반 마크다운

GitHub Flavored Markdown(GFM)을 지원합니다.

### 텍스트 서식

**굵게** 표시하면 하이라이트 배경이 적용됩니다 (원본 위키 스타일).

`인라인 코드`는 배경색이 적용됩니다.

### 인용문

> 인용문은 세리프 폰트(Noto Serif KR)로 표시됩니다.
> 좌측에 녹색 계열 테두리가 붙습니다.

### 코드 블록

```go
func main() {
    fmt.Println("Hello, akwiki!")
}
```

### 목록

- 순서 없는 목록
- 들여쓰기 지원
  - 하위 항목

1. 순서 있는 목록
2. 번호 자동

### 테이블

| 기능 | 설명 |
|------|------|
| 위키링크 | 페이지 간 양방향 연결 |
| 백링크 | 자동 역참조 추적 |
| TOC | 목차 자동 생성 |

---

이 구분선은 원본 위키처럼 `- - - § - - -` 텍스트로 표시됩니다.

### 이미지

```markdown
![대체 텍스트](이미지경로.png)
```

`public/` 디렉토리에 이미지를 넣으면 빌드 시 `assets/`로 복사됩니다.

## 날짜

`createdAt`과 `modifiedAt`는 frontmatter에 쓰지 않습니다. git 이력에서 자동 추출됩니다.

- 첫 커밋 날짜 → createdAt
- 마지막 커밋 날짜 → modifiedAt
- git 이력이 없으면 파일 시스템 시간을 사용합니다.
